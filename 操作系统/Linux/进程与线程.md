# 进程与线程

## 进程是什么

进程是资源分配的最小单位。在 Linux 内核中，进程由 `task_struct`（进程描述符）表示：

```c
struct task_struct {
    long              state;           // 进程状态
    struct mm_struct  *mm;             // 虚拟内存结构体
    pid_t             pid;             // 进程号
    struct task_struct __rcu *parent;  // 指向父进程
    struct list_head  children;        // 子进程列表
    struct fs_struct  *fs;             // 文件系统信息
    struct files_struct *files;        // 打开的文件指针数组
};
```

其中 `mm` 指向进程的虚拟内存，`files` 指向该进程打开的所有文件的指针数组。

## 文件描述符

文件描述符是 `files` 指针数组的索引（非负整数），0 为标准输入、1 为标准输出、2 为标准错误。

Linux 中"一切皆文件"，设备、进程、socket 套接字等都可通过文件描述符读写。

## 线程是什么

Linux 内核未将线程与进程区别对待，线程和进程都用 `task_struct` 表示，区别在于共享的数据区域不同：

| | 进程（fork） | 线程（pthread） |
|:---|:---|:---|
| mm 结构 | 拷贝副本，独立 | 共享父进程地址空间 |
| files 结构 | 拷贝副本，独立 | 共享文件描述符表 |
| 创建开销 | 较大（但 Linux 用 COW 优化） | 较小 |

## 进程 vs 线程 vs 协程

| 维度 | 进程 | 线程 | 协程 |
|:---|:---|:---|:---|
| 资源分配 | CPU 资源分配基本单位 | 独立运行调度的基本单位 | 用户态轻量级线程 |
| 资源 | 独立资源空间 | 共享进程内资源 | 共享线程资源 |
| 切换开销 | 大 | 中 | 小（用户态切换） |
| 多核利用 | 可 | 可 | 需配合进程 |

## 进程上下文切换 vs 线程上下文切换

**进程上下文切换**：需保存虚拟内存、栈、全局变量等用户空间资源，以及内核堆栈、寄存器等内核空间资源。

**线程上下文切换**：主要保存程序计数器、栈信息、寄存器等运行状态，因共享进程内存空间，无需大量内存管理。

## 进程切换步骤

1. 保存处理机上下文（程序计数器和其他寄存器）
2. 更新 PCB 信息
3. 将进程 PCB 移入相应队列
4. 选择另一个进程执行，更新其 PCB
5. 更新内存管理数据结构
6. 恢复处理机上下文

## 进程的阻塞

正在执行的进程因期待的某些事件未发生（如请求系统资源失败、等待 IO 完成等），由系统自动执行阻塞原语，使自己从运行态变为阻塞态。进程的阻塞是主动行为，阻塞时不占用 CPU 资源。

## 创建进程的方式

| 方式 | 说明 |
|:---|:---|
| **fork()** | 子进程完全复制父进程资源，有独立 task_struct 和 pid |
| **vfork()** | 子进程与父进程共享地址空间，子进程修改变量会影响父进程 |
| **clone()** | 有选择地复制父进程资源，通过 clone_flags 控制复制内容 |

三个调用最终都通过 `do_fork()` 实现。

## 进程调度算法

| 算法 | 说明 | 特点 |
|:---|:---|:---|
| 先来先服务（FCFS） | 按到达顺序调度 | 简单，无法充分利用 CPU |
| 最短进程优先（SJF） | 执行时间短的先执行 | 最小化平均等待时间，需预知执行时间 |
| 时间片轮转（RR） | 每个进程执行一个时间片 | 简单，响应时间和周转时间长 |
| 最高优先级（HPF） | 优先级高的先执行 | 可能出现饥饿 |
| 多级反馈队列（MFQ） | 多个队列不同时间片 | 兼顾长短进程，实现较复杂 |

## 进程间通信（IPC）

| 方式 | 特点 | 适用场景 |
|:---|:---|:---|
| **无名管道（Pipe）** | 半双工，单向，用于父子进程 | 简单消息传递 |
| **命名管道（FIFO）** | 支持双向，无关进程可通信 | 跨进程单向传输 |
| **信号（Signal）** | 异步通知，信息量小 | 进程中断/终止通知 |
| **消息队列** | 以消息形式传递结构化数据 | 复杂多进程通信 |
| **共享内存** | 最快的方式，直接读写公共内存 | 大量数据高速共享 |
| **信号量（Semaphore）** | 用于同步和互斥，不传递数据 | 控制共享资源访问 |
| **套接字（Socket）** | 支持网络和本地通信 | 跨主机分布式通信 |
| **文件映射（MMF）** | 将文件映射到进程地址空间 | 多进程共享文件数据 |
| **DBus** | Linux 桌面系统总线通信 | 桌面应用消息传递 |
| **RPC** | 远程过程调用，隐藏通信细节 | 微服务分布式调用 |
