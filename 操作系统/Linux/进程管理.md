# 进程管理

## 查看进程

### ps

查看某个时间点的进程信息。

```bash
ps -l                    # 查看自己的进程
ps aux                   # 查看系统所有进程
ps aux | grep threadx    # 查看特定进程
```

### pstree

查看进程树。

```bash
pstree -A                # 查看所有进程树
```

### top

实时显示进程信息。

```bash
top -d 2                 # 每 2 秒刷新一次
```

### netstat

查看占用端口的进程。

```bash
netstat -anp | grep port
```

### free

查看内存使用状况。

### 进程状态

| 状态 | 说明 |
|:---:|:---|
| R | running / runnable，正在执行或在执行队列中 |
| D | uninterruptible sleep，不可中断阻塞（通常为 IO） |
| S | interruptible sleep，可中断阻塞（等待事件完成） |
| Z | zombie，僵死，已终止但未被父进程获取信息 |
| T | stopped，被作业控制信号结束或正在被追踪 |

## SIGCHLD

当一个子进程状态改变（停止、继续或退出）时：
- 父进程收到 SIGCHLD 信号
- `waitpid()` 或 `wait()` 调用返回

子进程退出时进程描述符不会立即释放，父进程通过 `wait()` 和 `waitpid()` 获取子进程信息后才释放。

### wait()

```c
pid_t wait(int *status)
```
父进程调用 `wait()` 会一直阻塞，直到收到子进程退出的 SIGCHLD 信号。成功返回子进程 PID，失败返回 -1。

### waitpid()

```c
pid_t waitpid(pid_t pid, int *status, int options)
```
比 `wait()` 多出 `pid`（指定子进程 ID）和 `options`（WNOHANG 可使调用非阻塞）两个参数。

## 孤儿进程

父进程退出后，其子进程成为孤儿进程，被 init 进程（PID=1）收养并完成状态收集。孤儿进程不会对系统造成危害。

## 僵尸进程

子进程退出时，若父进程未调用 `wait()` 或 `waitpid()`，进程描述符仍保留在系统中，状态为 Z（zombie）。大量僵尸进程会耗尽系统进程号。

**解决方案**：杀死僵尸进程的父进程，使其变成孤儿进程，由 init 进程收养并释放资源。
