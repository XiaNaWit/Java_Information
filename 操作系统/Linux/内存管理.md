# 内存管理

## 物理内存与虚拟内存

| 概念 | 说明 |
|:---|:---|
| **物理内存** | 真实的内存硬件（如 4G/8G/16G 内存条） |
| **虚拟内存** | 操作系统为每个进程提供的独立、连续的虚拟地址空间 |

对于 4G 内存的 Linux 系统，虚拟内存也为 4G（通常 1G 内核空间 + 3G 用户空间）。每个进程认为自己占据全部地址空间，但仅在真正使用时才分配物理内存。

## 内存映射

### 映射表

内核为每个进程维护一张页表，记录虚拟地址到物理地址的映射关系。当 CPU 运行进程 A 时，虚拟地址空间切换为进程 A 的映射表；切换到进程 B 时，映射表随之切换。

### 缺页异常

当进程访问的虚拟地址在页表中查不到时，系统产生**缺页异常**，进入内核空间分配物理内存、更新页表，再返回用户空间恢复进程执行。

### 多级页表

Linux 采用四级页表管理内存，将内存分成区块，通过区块索引和偏移量定位。由于虚拟内存通常只使用很少一部分，多级页表只保存使用中的区块，大大减少页表项数。

### 内存映射工作流程

1. 进程访问虚拟地址
2. MMU 查询页表获取物理地址
3. 若页表无映射 → 缺页异常
4. 内核分配物理内存，更新页表
5. 恢复进程执行

## 页面置换算法

当物理内存不足时，需选择一个页面置换出去以腾出空间。

| 算法 | 策略 | 特点 |
|:---|:---|:---|
| **OPT（最优置换）** | 置换未来最长时间内不再被访问的页面 | 理论上最优，无法实现 |
| **FIFO** | 置换最早被调入的页面 | 简单，可能出现 Belady 异常 |
| **LRU**（最近最少使用） | 置换最近一段时间内最少被访问的页面 | 性能好，实现开销大 |
| **Clock**（时钟置换） | 类似 FIFO，使用访问位辅助决策 | 近似 LRU 的效率，实现简单 |
| **NFU**（最不经常使用） | 置换被访问次数最少的页面 | 实现简单，但可能"记仇" |
