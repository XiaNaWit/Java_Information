# 操作系统基础

## 用户态和内核态

为了区分不同程序的权限，操作系统引入了两种运行级别：**用户态**和**内核态**。

| | 用户态 | 内核态 |
|:---|:---|:---|
| 特权级 | 最低 | 最高 |
| 可访问范围 | 只能访问用户空间 | 可访问内核空间所有资源 |
| 典型程序 | 用户应用程序 | 操作系统内核、驱动 |

### 用户态到内核态的切换方式

**1. 系统调用**：用户进程主动申请操作系统服务（如 read/write），核心是触发中断（Linux 的 `int 80h`）。

**2. 异常**：CPU 执行用户态程序时发生异常（如缺页异常），切换到内核态处理。

**3. 外围设备中断**：硬件完成操作后发中断信号，CPU 暂停当前程序转去执行中断处理程序（如硬盘读写完成）。

### 切换步骤

1. 从当前进程描述符提取内核栈的 ss0 和 esp0 信息
2. 使用内核栈保存当前进程的 cs、eip、eflags、ss、esp 信息（用户栈 → 内核栈）
3. 将中断处理程序的 cs、eip 装入寄存器，开始执行内核程序

## 轮询和中断

| 方式 | 说明 | 优点 | 缺点 |
|:---|:---|:---|:---|
| **轮询（Polling）** | 主动循环检查状态是否变化 | 简单易懂 | 占用大量 CPU，效率低 |
| **中断（Interrupt）** | 外部事件触发时被动处理 | 减少 CPU 占用，效率高 | 实现较复杂 |

实际系统中两者常结合使用：中断处理高优先级事件，CPU 空闲时轮询检查其他设备。

## CPU 100% 排查

### 排查步骤

1. 执行 `top` 查看 CPU 最高的进程 PID
2. 执行 `top -Hp <进程号>` 查看进程中所有线程的 CPU 占用
3. 执行 `printf "%x\n" <线程号>` 将线程号转为十六进制
4. 执行 `jstack <进程号> \| grep <线程ID>` 查看线程堆栈

### 线程状态

| 状态 | 说明 |
|:---|:---|
| NEW | 未启动 |
| RUNNABLE | 运行中（可能有 locked 字样，表示获得某把锁） |
| BLOCKED | 阻塞，等待监视器锁 |
| WAITING | 无限期等待（park/wait/sleep/join） |
| TIMED_WAITING | 有时限等待 |
| TERMINATED | 已退出 |

关注 `deadlock` 情况，必要时使用 `jmap -dump <pid>` 导出堆转储文件分析。

## sudo

`sudo` 允许一般用户以 root 权限执行命令，需在 `/etc/sudoers` 中配置。

## GNU 与开源协议

GNU 计划目标创建完全自由的操作系统，内容以 GPL 方式发布。GPL 包含：
- 以任何目的运行此程序的自由
- 再复制的自由
- 改进并公开发布改进的自由
