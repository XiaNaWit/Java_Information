## 外观（Facade）

### Intent

提供了一个统一的接口，用来访问子系统中的一群接口，从而让子系统更容易使用。

### Implementation

观看电影需要操作很多电器，使用外观模式实现一键看电影功能。

```java
// 子系统：多个独立的电器类
public class TV {
    public void turnOn() {
        System.out.println("turnOn TV");
    }
    public void turnOff() {
        System.out.println("turnOff TV");
    }
}

public class SoundSystem {
    public void turnOn() {
        System.out.println("turnOn SoundSystem");
    }
    public void setVolume(int level) {
        System.out.println("setVolume to " + level);
    }
}

public class DVDPlayer {
    public void play(String movie) {
        System.out.println("play " + movie);
    }
    public void stop() {
        System.out.println("stop DVD");
    }
}

public class Light {
    public void dim(int level) {
        System.out.println("dim light to " + level);
    }
}
```

```java
// 外观类：一键控制所有电器
public class HomeTheaterFacade {
    private TV tv = new TV();
    private SoundSystem sound = new SoundSystem();
    private DVDPlayer dvd = new DVDPlayer();
    private Light light = new Light();

    public void watchMovie(String movie) {
        light.dim(10);
        tv.turnOn();
        sound.turnOn();
        sound.setVolume(20);
        dvd.play(movie);
    }

    public void endMovie() {
        dvd.stop();
        tv.turnOff();
        System.out.println("end movie");
    }
}
```

```java
public class Client {
    public static void main(String[] args) {
        HomeTheaterFacade facade = new HomeTheaterFacade();
        facade.watchMovie("Inception");
        // dim light to 10
        // turnOn TV
        // turnOn SoundSystem
        // setVolume to 20
        // play Inception
        facade.endMovie();
        // stop DVD
        // turnOff TV
        // end movie
    }
}
```

### 外观模式与适配器模式的区别

| 模式 | 目的 | 举例 |
|:---|:---|:---|
| **外观模式** | 为子系统提供简化接口，**简化使用** | HomeTheaterFacade 一键看电影 |
| **适配器模式** | 将一个接口转换成客户端期望的另一个接口，**解决接口不兼容** | 将 220V 转为 5V 给手机充电 |

外观模式没有对子系统进行功能上的改造（不改接口），只是将多个子系统的调用封装成一个更易用的接口；适配器模式则是对单个类的接口进行转换。

### 设计原则

**最少知识原则**：只和你的密友谈话。客户对象所需要交互的对象应当尽可能少。外观类正是客户与众多子系统之间的"密友"，客户只需要和外观交互即可，无需了解多个子系统。

### JDK

- [javax.faces.context.FacesContext](https://docs.oracle.com/javaee/7/api/javax/faces/context/FacesContext.html)：内部使用了 Lifecycle、NavigationHandler 等多个子系统，对外提供统一的方法
- [org.springframework.jdbc.support.JdbcTransactionManager](https://docs.spring.io/spring-framework/docs/current/javadoc-api/org/springframework/jdbc/support/JdbcTransactionManager.html)：封装了 JDBC 相关操作
- [SLF4J](http://www.slf4j.org/)：对各种日志框架（Logback、Log4j）提供统一的外观接口
