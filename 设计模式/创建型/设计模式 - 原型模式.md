## 6. 原型模式（Prototype）

### Intent

使用原型实例指定要创建对象的类型，通过复制这个原型来创建新对象。

### 浅拷贝与深拷贝

**浅拷贝**：只复制基本类型的值，引用类型字段复制的是引用地址，新旧对象共享引用对象。

**深拷贝**：复制对象及其所有引用的对象，新旧对象完全独立。

| 特性 | 浅拷贝 | 深拷贝 |
|:---|:---|:---|
| 基本类型字段 | 复制值，互不影响 | 复制值，互不影响 |
| 引用类型字段 | 复制引用，共享对象 | 递归复制，完全独立 |
| 性能 | 较快 | 较慢 |
| 实现难度 | 简单 | 复杂 |

### Implementation

**方式一：实现 Cloneable 接口（浅拷贝）**

```java
public class ConcretePrototype implements Cloneable {

    private String filed;
    private int[] data;  // 引用类型字段

    public ConcretePrototype(String filed) {
        this.filed = filed;
        this.data = new int[]{1, 2, 3};
    }

    @Override
    public ConcretePrototype clone() {
        try {
            return (ConcretePrototype) super.clone();
        } catch (CloneNotSupportedException e) {
            throw new RuntimeException(e);
        }
    }

    @Override
    public String toString() {
        return "filed=" + filed + ", data=" + Arrays.toString(data);
    }
}
```

```java
public class Client {
    public static void main(String[] args) {
        ConcretePrototype prototype = new ConcretePrototype("abc");
        ConcretePrototype clone = prototype.clone();
        System.out.println(clone.toString());  // filed=abc, data=[1, 2, 3]
        System.out.println(prototype == clone);          // false
        System.out.println(prototype.data == clone.data); // true（浅拷贝，引用相同）
    }
}
```

**方式二：通过序列化实现深拷贝**

利用 Java 对象序列化机制，将对象写入流再读回，实现完全独立的深拷贝：

```java
public class DeepClonePrototype implements Serializable {

    private String filed;
    private int[] data;

    public DeepClonePrototype(String filed) {
        this.filed = filed;
        this.data = new int[]{1, 2, 3};
    }

    public DeepClonePrototype deepClone() {
        try {
            // 序列化
            ByteArrayOutputStream bos = new ByteArrayOutputStream();
            ObjectOutputStream oos = new ObjectOutputStream(bos);
            oos.writeObject(this);
            // 反序列化
            ByteArrayInputStream bis = new ByteArrayInputStream(bos.toByteArray());
            ObjectInputStream ois = new ObjectInputStream(bis);
            return (DeepClonePrototype) ois.readObject();
        } catch (Exception e) {
            throw new RuntimeException(e);
        }
    }
}
```

> 注意：所有被引用的对象也必须实现 Serializable，否则会抛出 NotSerializableException。

### 适用场景

- 创建对象成本较大（如需要复杂的数据库查询或网络请求），且对象间只有少量差异时
- 对象需要避免被外部修改时，可返回一个拷贝副本（保护性复制）
